Prerequisite:

CircleCI needs the RELEASE_PROFILE_BBOTTEMA context with:

- CENTRAL_USERNAME
- CENTRAL_PASSWORD
- GPG_PASSPHRASE

The project publishes through Sonatype Central Portal using server id `central`
from `.circleci/maven-central-settings.xml`.

To release:

1. Make sure release notes and GitHub issue/milestone bookkeeping are complete.
2. Push the release-ready commit to `master`.
3. Wait for the CircleCI build-and-test job to pass.
4. Approve the appropriate deploy job in CircleCI:
   - deploy-patch-version
   - deploy-minor-version
   - deploy-major-version
   - deploy-as-is-version
5. Confirm the released version appears in Maven Central metadata.
6. Close the GitHub milestone and update GitHub Releases if needed.

Local deploy equivalent, when needed:

    mvn -s .circleci/maven-central-settings.xml clean deploy -DperformRelease=true -DskipTests -Dspotbugs.skip=true -Denforcer.skip=true -Djacoco.skip=true

Maven settings server:

    <server>
      <id>central</id>
      <username>${env.CENTRAL_USERNAME}</username>
      <password>${env.CENTRAL_PASSWORD}</password>
    </server>

GPG profile:

    <profiles>
      <profile>
        <id>gpg</id>
        <properties>
          <gpg.executable>gpg</gpg.executable>
          <gpg.passphrase>${env.GPG_PASSPHRASE}</gpg.passphrase>
        </properties>
      </profile>
    </profiles>
    <activeProfiles>
      <activeProfile>gpg</activeProfile>
    </activeProfiles>
